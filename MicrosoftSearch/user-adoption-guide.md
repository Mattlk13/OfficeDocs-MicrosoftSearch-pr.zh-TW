# <a name="user-adoption-guide"></a>使用者採用指南

這項資訊適用於 Microsoft Search 系統管理員、變更管理領導者以及企業擁有者。
  
若要推出並確保整個組織成功採用 Microsoft Search，建議您使用這四個階段程序。
  
## <a name="stage-1-stakeholders"></a>階段 1：專案關係人

若要確保 Microsoft Search 順暢推出並增加使用者採用率，請找出可協助支援並推動程序的人員。
  
下表顯示團隊階層與每個人員可貢獻的部分。 
  
 **使用者採用團隊角色及責任**
  
|**角色**|**責任**|**部門**|
|:-----|:-----|:-----|
|執行發起人  <br/> |瞭解貴組織 Microsoft 搜尋的高層遠景和價值。  <br/> 為 IT/業務小組提供領導支援。  <br/> 分配資源。  <br/> 直接與員工交流。  <br/> |執行領導  <br/> |
|企業主  <br/> |識別產品和使用者案例。  <br/> 決定部門和組織的成功計量。  <br/> |任何  <br/> |
|變更管理團隊  <br/> |確定產品值很容易理解。  <br/> 建立「推出」的計畫。  <br/> |IT  <br/> |
|風雲人物  <br/> |將產品 Evangelize 至對等和團隊。  <br/> |任何人 (員工)  <br/> |
|搜尋系統管理員  <br/> |識別使用者採用的核心小組成員。  <br/> 管理 Microsoft 搜尋，包含經驗和持續管理的外觀與風格。  <br/> [磁片磁碟機討論] 顯示 Microsoft 搜尋的值。  <br/> |IT  <br/> |
|編輯者  <br/> |在 Microsoft 365 系統[管理中心](https://admin.microsoft.com)建立內容。  <br/> |IT/任何人  <br/> |
   
## <a name="stage-2-scenarios"></a>階段 2：案例

若要確保使用者採用，務必判斷和定義使用者案例。 這應該包含使用者如何使用 Microsoft Search 來尋找所需的資訊、回答問題，並更快速地完成工作的實際範例。 為了評估您的推出和使用者採用的成功度，請務必在開始之前，先決定您將使用的指標。
  
### <a name="understand-what-your-users-need-and-want"></a>了解您的使用者需要及想要什麼
  
談到尋找資訊和取得解答，了解使用者的需求和困難點是很重要的。 您可以使用定量方法，就像[Office 365](https://Office.com)和[SharePoint](https://products.office.com/sharepoint/collaboration)搜尋使用方式或定性方法（如調查和使用者焦點群組）的資料，以取得這類資料。 這將有助於判斷 Microsoft Search 何時、何處以及如何提供價值。
  
針對整個組織的不同角色和團隊，請考慮：
  
- 現有使用者搜尋資訊時的困難。
    
- 常用的應用程式、工具、網站和其他資訊來源。
    
- 使用者對於搜尋資訊想要看到什麼。
    
- Microsoft Search 能為組織帶來的優點。 如需詳細資訊，請參閱[Microsoft 搜尋概述](overview-microsoft-search.md)。
    
選擇對您的使用者具有最大正面影響的使用者案例。
  
 ### <a name="identify-success-metrics"></a>找出成功指標
  
為了協助您符合推行目標，並展現對組織的正面影響，請務必在開始之前，先決定關鍵的成功指標。 將您的目標保持簡單且可度量，並定期評估效能，以便在需要時您可以快速回應。
  
1. **變更管理：** 決定如何測量人員對於 Microsoft Search 的意識和其相關通訊的成功程度。 
    
2. **使用率：** 您可以利用來自深入解析儀表板的資料，包括每日及每月作用中使用者數 (DAU 和 MAU)，獲得最多檢視次數的內容類型，以及最常見的搜尋關鍵字，以隨著時間推移測量使用率。 如需詳細資訊，請參閱[Microsoft 搜尋 Insights 儀表板報告](./usage-reports.md)。
    
3. **使用者滿意度：** 使用您從問卷和使用者焦點座談所收集的資料，來判斷比較基準。 淨滿意度 (NSAT)，以及淨推薦分數 (NPS) 也可以提供實用的度量單位。 不論您使用的方法為何，請保持一致，這樣您就能看到隨時間的影響。 
    
4. **其他測量方式：** 針對支援要求使用歷程記錄和/或最新資料，以及搜尋所耗費的時間量。 
    
## <a name="stage-3-readiness"></a>階段 3：整備

投入時間在準備推出 Microsoft Search，可為組織帶來正面影響。 Microsoft 提供 <a href="https://go.microsoft.com/fwlink/?linkid=2114710">採用套件</a> ，其中包含為內部通訊網站上的共用建立的宣告電子郵件範本、輕量、視覺內容，以及訓練簡報。 還有一部<a href="https://resources.techcommunity.microsoft.com/microsoft-search/#adoption">Microsoft 搜尋資源中心</a>，您可以在其中尋找其他訓練和採用資源，以協助讓您的 Microsoft 搜尋部署成功。
  
### <a name="product-and-content-readiness"></a>產品與內容整備
  
系統管理員會直接影響使用者的搜尋體驗，並為組織決定 Microsoft Search 的外觀。 這包括選擇想要呈現給使用者的結果類型。 這些結果可能包括連結 (也稱為書籤)、文件、網站、人員、群組、位置、交談和其他工作場所資源。
  
如需詳細資訊，請參閱[設定 Microsoft Search](setup-microsoft-search.md)。
  
編輯者為能在其專業技術領域協助您管理書籤和問與答的主題專家。 編輯者應該建立可解決您稍早找出之使用者案例的內容。
  
如需詳細資訊，請參閱[規劃內容](plan-your-content.md)。
  
### <a name="signing-in"></a>登入
  
Microsoft 搜尋使用者必須使用工作或學校帳戶登入[Bing](https://Bing.com) 。 若要讓您的使用者的登入盡可能順暢地：
  
- 若要確定是否已正確設定[Azure Active Directory](/azure/active-directory/)設定，請稍候的使用者嘗試登入。 如需詳細資訊，請參閱[測試單一登入](test-single-sign-on.md)。
    
- 將公司管理裝置的瀏覽器預設值設定為 Bing。 如需詳細資訊，請參閱[設定預設瀏覽器](/deployedge/edge-default-browser)。
    
- 透過「群組原則」，為[Microsoft Edge](https://www.microsoft.com/windows/microsoft-edge)和[Google Chrome](https://www.google.com)使用者部署搜尋擴充。 如需詳細資訊，請參閱[Microsoft 搜尋概述](overview-microsoft-search.md)。
    
### <a name="draft-communications"></a>草稿通訊
  
使用您實際的案例來向使用者說明 Microsoft Search，以及它如何協助他們節省時間。 您的變更管理團隊可協助建立推出策略，以及為使用者整備準備資料和活動。
  
開始規劃您的組織的通訊及活動：
  
- 將橫幅、海報或 leaflets 放在明顯的位置。
    
- 個人產品示範或 booths。
    
- 本機和遠端小組的線上事件。
    
- 讓您的執行贊助者主持城鎮廳會議或傳送宣告電子郵件。
    
- 分享使用 Microsoft 搜尋的自助檔。
    
所有通訊都應教育使用者如何登入[Bing](https://Bing.com) ，以尋找工作結果。
  
### <a name="engage-partner-teams"></a>吸引合作夥伴團隊
  
[Microsoft 365](https://www.microsoft.com/microsoft-365)的整數部分，Microsoft 搜尋遵循 GDPR 和第 C 層的指導方針。 視您的組織而定，您可能需要與隱私權和安全性小組共用 Microsoft 搜尋相關資訊。 如需詳細資訊，請參閱[Bing 專用 Microsoft 搜尋的安全性和隱私權](security-for-search.md)。
  
請確認您的技術支援中心知道 Microsoft Search 的相關特性與功能，以及使用者的登入需求，使他們可以更有效率地支援使用者。 如需詳細資訊，請參閱瞭解 Microsoft 365 和[FAQs](faqs.md)[中的 Microsoft 搜尋](https://support.office.com/article/Learn-about-Microsoft-Search-in-Microsoft-365-b8bf5a2c-7515-40a9-9a6a-b8ed382c86bc)。
  
### <a name="evaluate-with-pilot-users"></a>使用試驗使用者來評估
  
從一小組使用者開始來評估 Microsoft Search 的影響，並及早收集意見反應。 如此可讓您有足夠的時間變更內容，更新您的通訊，以及為廣泛推出設定預期。 搜尋進階使用者或早期採用者是針對此目的的良好對象。
  
若要協助找出試驗使用者：
  
- 與貴組織內的商務贊助者合作。
    
- 識別一組前期消費者。
    
- 識別執行大量企業搜尋的群組。
    
- 識別位於其他地理位置的使用者，包括國際。
    
要求試驗使用者使用 Microsoft Search 中的意見反應選項或透過簡單的問卷來提供意見。
  
要詢問的問卷問題： 
  
- 您了解如何登入並使用 Microsoft Search 嗎？
    
- 它可如預期運作嗎？
    
- 您喜歡什麼功能？
    
- 您可以找到工作結果嗎？
    
- 工作結果有幫助嗎？
    
- 關於 Microsoft Search 的通訊是否清楚且易於理解？
    
這是開始測量階段 2 中所找出指標有效性的絕佳時機。
  
開始試驗階段時，請考慮與更廣泛的組織分享 Microsoft Search 的快速預覽。
  
## <a name="stage-4-roll-out"></a>階段 4：推出

每個組織對產品推出的想法不同。 以下是要考量的一些事項。
  
### <a name="broad-deployment"></a>廣泛部署
  
當您準備好時，可讓組織中的所有使用者都能存取 Microsoft 搜尋，或使用分階段的方法來更緩慢地執行。 
  
### <a name="generate-awareness"></a>產生意識
  
在組織中提倡 Microsoft Search 以建立吸引力：
  
- 召集試驗使用者來提供端對端協助和專業技術。
    
- 遵循您在階段 3 中規劃的通訊和活動。
    
### <a name="ongoing-communication"></a>持續通訊
  
在前幾週和幾個月，保持規律通訊，以鼓勵參與 Microsoft Search。 這可讓使用者收到通知，並讓新使用者參與。 每個通訊應該說明 Microsoft Search 為何對他們的工作有幫助。
  
持續傳送通訊，強調可嘗試的功能和搜尋，以及使用者的成功故事。
  
### <a name="gather-feedback-and-improve"></a>收集意見反應並改善
  
支援使用者，並透過聆聽他們的意見並根據他們的意見反應採取行動，讓他們知道您的重視。
  
使用深入解析儀表板來查看產品和內容的執行情況。 利用此資訊來更新書籤、問與答，以及其他通訊。 這也能協助您：
  
- 改進內容品質：查看標題、描述及 URLs、優化關鍵字、移除過時的資訊，以及新增資訊。
    
2. 決定最常使用或不經常使用的功能。 
    
3. 執行調查、焦點群組和意見反應，以瞭解使用者對 Microsoft 搜尋、提高採用程度及識別改進區域的看法。
    
教育和意見反應是具重複性的程序。 在使用者採用 Microsoft 搜尋時，可能需要變更內容和通訊。
