# <a name="user-adoption-guide"></a>使用者採用指南

這項資訊適用於 Microsoft Search 系統管理員、變更管理領導者以及企業擁有者。
  
若要推出並確保整個組織成功採用 Microsoft Search，建議您使用這四個階段程序。
  
## <a name="stage-1-stakeholders"></a>階段 1：專案關係人

若要確保 Microsoft Search 順暢推出並增加使用者採用率，請找出可協助支援並推動程序的人員。
  
下表顯示團隊階層與每個人員可貢獻的部分。 
  
 **使用者採用團隊角色及責任**
  
|**角色**|**責任**|**部門**|
|:-----|:-----|:-----|
|執行發起人  <br/> |了解 Microsoft Search 對您的組織的高層觀點與價值  <br/> 對 IT/業務團隊提供領導支援  <br/> 配置資源  <br/> 直接與員工溝通  <br/> |執行領導  <br/> |
|企業主  <br/> |找出產品與使用者案例  <br/> 決定部門與組織的成功指標  <br/> |任何  <br/> |
|變更管理團隊  <br/> |確保產品價值易於了解  <br/> 建立推出計劃  <br/> |IT  <br/> |
|風雲人物  <br/> |向同儕與團隊宣傳產品  <br/> |任何人 (員工)  <br/> |
|搜尋系統管理員  <br/> |找出使用者採用的核心團隊成員  <br/> 管理 Microsoft Search，包括體驗的外觀及操作並持續管理  <br/> 促進討論以展現 Microsoft Search 的價值  <br/> |IT  <br/> |
|編輯者  <br/> |在系統管理入口網站中建立內容  <br/> |IT/任何人  <br/> |
   
## <a name="stage-2-scenarios"></a>階段 2：案例

若要確保使用者採用，務必判斷和定義使用者案例。 這應該包含使用者如何使用 Microsoft Search 來尋找所需的資訊、回答問題，並更快速地完成工作的實際範例。 為了評估您的推出和使用者採用的成功度，請務必在開始之前，先決定您將使用的指標。
  
 **了解您的使用者需要及想要什麼**
  
談到尋找資訊和取得解答，了解使用者的需求和困難點是很重要的。 您可以使用量化方法 (例如來自 Office 365 或 SharePoint 搜尋使用率的資料)，或是質化方法 (例如問卷和使用者焦點座談) 來取得這項資料。 這將有助於判斷 Microsoft Search 何時、何處以及如何提供價值。
  
針對整個組織的不同角色和團隊，請考慮：
  
- 現有使用者搜尋資訊時的困難。
    
- 常用的應用程式、工具、網站和其他資訊來源。
    
- 使用者對於搜尋資訊想要看到什麼。
    
- Microsoft Search 能為組織帶來的優點。 如需詳細資訊，請參閱[為什麼要使用 Microsoft Search](why-microsoft-search.md)。
    
選擇對您的使用者具有最大正面影響的使用者案例。
  
 **找出成功指標**
  
為了協助您符合推行目標，並展現對組織的正面影響，請務必在開始之前，先決定關鍵的成功指標。 讓您的目標保持簡單並可測量，以及定期評估效能，使得您在必要時可快速回應。
  
1. **變更管理：** 決定如何測量人員對於 Microsoft Search 的意識和其相關通訊的成功程度。 
    
2. **使用率：** 您可以利用來自深入解析儀表板的資料，包括每日及每月作用中使用者數 (DAU 和 MAU)，獲得最多檢視次數的內容類型，以及最常見的搜尋關鍵字，以隨著時間推移測量使用率。 如需詳細資訊，請參閱[使用深入解析儀表板](get-insights.md)。
    
3. **使用者滿意度：** 使用您從問卷和使用者焦點座談所收集的資料，來判斷比較基準。 淨滿意度 (NSAT)，以及淨推薦分數 (NPS) 也可以提供實用的度量單位。 不論使用何種方法都應保持一致，使得您可以隨著時間推移查看影響。 
    
4. **其他測量方式：** 針對支援要求使用歷程記錄和/或最新資料，以及搜尋所耗費的時間量。 
    
## <a name="stage-3-readiness"></a>階段 3：整備

投入時間在準備推出 Microsoft Search，可為組織帶來正面影響。 Microsoft 提供<a href="https://resources.techcommunity.microsoft.com/microsoft-search-in-bing-adoption-kit/">採用套件</a> (英文)，包含規劃指引、可自訂的電子郵件宣告、問卷範本、執行簡報等。 此外，還有 <a href="https://resources.techcommunity.microsoft.com/microsoft-search/#adoption">Microsoft 搜尋資源中心</a> (英文)，提供 IT 實作者其他可協助成功推出 Microsoft 搜尋的訓練和採用資源。
  
 **產品與內容整備**
  
系統管理員會直接影響使用者的搜尋體驗，並為組織決定 Microsoft Search 的外觀。 這包括選擇想要呈現給使用者的結果類型。 這些結果可能包括連結 (也稱為書籤)、文件、網站、人員、群組、位置、交談和其他工作場所資源。
  
如需詳細資訊，請參閱[設定 Microsoft Search](setup-microsoft-search.md)。
  
編輯者為能在其專業技術領域協助您管理書籤和問與答的主題專家。 編輯者應該建立可解決您稍早找出之使用者案例的內容。
  
如需詳細資訊，請參閱[規劃內容](plan-your-content.md)。
  
 **登入**
  
Microsoft Search 使用者必須使用公司或學校帳戶來登入 Bing。 若要讓您的使用者的登入盡可能順暢地：
  
- 請確定已正確設定 Azure Active Directory 的組態，有一些使用者嘗試登入。 如需詳細資訊，請參閱[測試單一登入](test-single-sign-on.md)。
    
- 將公司管理裝置的瀏覽器預設值設定為 Bing。 如需詳細資訊，請參閱[設定預設瀏覽器](set-default-browser.md)。
    
- 透過群組原則為 Edge 和 Chrome 使用者部署搜尋擴充功能。 如需詳細資訊，請參閱[系統管理員入口網站工具](admin-portal-tools.md)。
    
 **草稿通訊**
  
使用您實際的案例來向使用者說明 Microsoft Search，以及它如何協助他們節省時間。 您的變更管理團隊可協助建立推出策略，以及為使用者整備準備資料和活動。
  
開始規劃您的組織的通訊及活動：
  
- 將橫幅、海報或傳單放在明顯位置
    
- 面對面的產品示範或展示
    
- 當地和遠端團隊的線上活動
    
- 讓執行發起人主持全體會議或傳送公告電子郵件
    
- 分享有關使用 Microsoft Search 的自助文件
    
所有通訊內容應該教育使用者他們如何登入 Bing 來尋找工作結果。
  
 **吸引合作夥伴團隊**
  
Microsoft Search 為 Microsoft 365 不可或缺的一部分，它符合 GDPR 和 C 層級指導方針。 根據您的組織而定，您可能必須與您的隱私權和安全性團隊分享 Microsoft Search 的相關資訊。 如需詳細資訊，請參閱 [Microsoft Search 的安全性](security.md)。
  
請確認您的技術支援中心知道 Microsoft Search 的相關特性與功能，以及使用者的登入需求，使他們可以更有效率地支援使用者。 如需詳細資訊，請參閱[使用 Microsoft Search](use/about-microsoft-search.md) 與[常見問題集](faqs.md)。
  
 **使用試驗使用者來評估**
  
從一小組使用者開始來評估 Microsoft Search 的影響，並及早收集意見反應。 如此可讓您有足夠的時間變更內容，更新您的通訊，以及為廣泛推出設定預期。 搜尋進階使用者或早期採用者是針對此目的的良好對象。
  
若要協助找出試驗使用者：
  
- 與貴組織內的商業贊助商合作
    
- 找出一組早期採用者
    
- 找出進行大量企業搜尋的群組
    
- 找出位於其他地理位置的使用者 (包括國際)
    
要求試驗使用者使用 Microsoft Search 中的意見反應選項或透過簡單的問卷來提供意見。
  
要詢問的問卷問題： 
  
- 您了解如何登入並使用 Microsoft Search 嗎？
    
- 它可如預期運作嗎？
    
- 您喜歡什麼功能？
    
- 您可以找到工作結果嗎？
    
- 工作結果有幫助嗎？
    
- 關於 Microsoft Search 的通訊是否清楚且易於理解？
    
這是開始測量階段 2 中所找出指標有效性的絕佳時機。
  
開始試驗階段時，請考慮與更廣泛的組織分享 Microsoft Search 的快速預覽。
  
## <a name="stage-4-roll-out"></a>階段 4：推出

每個組織對產品推出的想法不同。 以下是要考量的一些事項。
  
 **廣泛部署**
  
當您準備好時，請允許組織中的所有使用者存取 Microsoft Search，或是使用分階段的方式慢慢推出。 
  
 **產生意識**
  
在組織中提倡 Microsoft Search 以建立吸引力：
  
- 召集試驗使用者來提供端對端協助和專業技術。
    
- 遵循您在階段 3 中規劃的通訊和活動。
    
 **持續通訊**
  
在前幾週和幾個月，保持規律通訊，以鼓勵參與 Microsoft Search。 這可讓使用者收到通知，並讓新使用者參與。 每個通訊應該說明 Microsoft Search 為何對他們的工作有幫助。
  
持續傳送通訊，強調可嘗試的功能和搜尋，以及使用者的成功故事。
  
 **收集意見反應並改善**
  
支援使用者，並透過聆聽他們的意見並根據他們的意見反應採取行動，讓他們知道您的重視。
  
使用深入解析儀表板來查看產品和內容的執行情況。 利用此資訊來更新書籤、問與答，以及其他通訊。 這也能協助您：
  
1. 改善內容品質：檢閱標題、描述及 URL，最佳化關鍵字，移除過時的資訊，並加入新的資訊
    
2. 判斷哪些功能最常或最少使用
    
3. 進行問卷、焦點座談和意見反應會議，了解使用者對於 Microsoft Search 的想法，增加採用率，並找出改善的領域
    
教育和意見反應是具重複性的程序。 當您的使用者採用 Microsoft Search，您可能需要對內容進行變更並通訊。
