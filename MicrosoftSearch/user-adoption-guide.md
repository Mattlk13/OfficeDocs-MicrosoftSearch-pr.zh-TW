# <a name="user-adoption-guide"></a>使用者採用指南

Microsoft Search 管理員、 變更管理負責人、 及企業擁有者適合這項資訊。
  
若要導入，並確定 Microsoft 搜尋整個組織成功採用，我們建議這四個階段程序。
  
## <a name="stage-1-stakeholders"></a>階段 1： 專案關係人

若要確保順暢導入並增加使用者採用 Microsoft Search，找出支援與磁碟機程序可協助的人員。
  
下表顯示小組階層和每個人的機率。 
  
 **使用者採用小組角色與責任**
  
|**角色**|**責任**|**部門**|
|:-----|:-----|:-----|
|贊助者  <br/> |了解的高層級的遠景與您的組織的 Microsoft 搜尋的值  <br/> 提供領導支援 IT / 業務團隊  <br/> 配置資源  <br/> 直接與員工通訊  <br/> |高階主管的領導  <br/> |
|企業主  <br/> |識別產品及使用者的案例  <br/> 決定部門和組織的成功的計量  <br/> |任何一個  <br/> |
|變更管理小組  <br/> |確定易懂的產品值  <br/> 建立的計劃推展  <br/> |IT  <br/> |
|冠軍  <br/> |宣傳同事及小組的產品  <br/> |任何 （人員）  <br/> |
|搜尋管理員  <br/> |識別使用者採用核心小組成員  <br/> 管理 Microsoft Search，包括體驗和持續管理的外觀與風格  <br/> 要顯示的 Microsoft 搜尋值的磁碟機討論區  <br/> |IT  <br/> |
|編輯者  <br/> |Admin 入口網站中建立的內容  <br/> |IT/任何  <br/> |
   
## <a name="stage-2-scenarios"></a>階段 2： 案例

若要確保使用者採用，很重要決定並定義使用者的案例。這些應該包括真實世界的人員可以使用範例 Microsoft Search 找到他們需要、 回答的問題，以及更快速完成工作的資訊。若要測量取出您 roll 是否成功與使用者採用也很重要決定您要使用之前的評量。
  
 **了解您的使用者需要以及想**
  
尋找資訊及快速回答而言，因為我們已經知道使用者的需求和設計師點是索引鍵。您可以使用類似來自 Office 365 與 SharePoint 搜尋流量或質化方法、 問卷和使用者焦點群組，以取得此資料的量化方法。這可協助您決定當 Microsoft Search，及如何可以提供值。
  
為各種角色和整個組織的小組，請考慮：
  
- 現有使用者設計師點時搜尋的資訊。
    
- 常用的應用程式、 工具、 網站及其他資訊來源。
    
- 哪些使用者想要查看專職時搜尋的資訊。
    
- 如何組織適用於 Microsoft Search 的優點。如需詳細資訊，請參閱[為什麼 Microsoft Search](why-microsoft-search.md)。
    
選擇將已為使用者有關有助於提升使用者案例。
  
 **識別成功計量**
  
為了協助您達成 roll 延展目標，並顯示有助於提升您的組織，很重要決定成功關鍵衡量標準開始之前。保持簡單且值得注意的目標與因此您可以使用回應快速視評估規則為基礎的效能。
  
1. **變更管理：** 決定如何將測量的 Microsoft Search 傳達和其相關的通訊成功。 
    
2. **Usage:** 您可以運用資料從前瞻儀表板包含每日及每月作用中的使用者 （DAU 和 MAU），會取得最檢視及最常用的內容類型搜尋關鍵字]，以衡量一段時間的使用方式。如需詳細資訊，請參閱 ＜ [Use 前瞻儀表板](get-insights.md)。
    
3. **使用者滿意度：** 使用您收集哪些調查及使用者焦點群組，以決定您的基準資料。Net 滿意度 (NSAT) 與 Net Promoter 分數 (NPS) 也可以提供實用的度量值。不論方法您可以使用，讓它保持一致，因此您可以看到一段時間的影響。 
    
4. **其他度量單位：** 歷程記錄和/或目前的資料可用於支援要求和搜尋所花費的時間量。 
    
## <a name="stage-3-readiness"></a>階段 3： 整備

演變準備 Microsoft Search 導入的時間會導致有助於提升您的組織。 
  
 **產品及內容的整備**
  
系統管理員直接影響使用者搜尋體驗，並決定組織的 Microsoft Search 的外觀。這包括選擇您要對使用者出現的結果類型。控制此開啟或關閉文件、 網站、 人員、 群組或交談的結果。
  
如需詳細資訊，請參閱[設定 Microsoft 搜尋](set-up-microsoft-search.md)。
  
編輯者是協助管理的專業知識其區域內的書籤和 Q&As 主題專家提供。編輯者應建立位址先前識別使用者案例的內容。
  
如需詳細資訊，請參閱[規劃您的內容](plan-your-content.md)。
  
 **登入**
  
Microsoft Search 使用者必須登入 Bing 與工作或學校帳戶。進行登入儘可能以一致的使用者：
  
- 請確定已正確設定 Azure Active directory 設定有一些使用者嘗試登入。如需詳細資訊，請參閱[測試單一登入](test-single-sign-on.md)。
    
- 設定瀏覽器的預設值為公司管理裝置 Bing 的。如需詳細資訊，請參閱[設定預設的瀏覽器](set-default-browser.md)。
    
- 部署搜尋延伸模組的邊緣和 Chrome 透過群組原則的使用者。如需詳細資訊，請參閱 ＜[管理入口網站的工具](admin-portal-tools.md)。
    
 **草稿通訊**
  
使用您的實際情況告訴使用者有關 Microsoft Search 及如何它將會協助他們節省時間。變更管理小組可協助建立導入策略和準備使用者整備附隨品和活動。
  
開始規劃通訊和您的組織的事件：
  
- 橫幅、 海報 （英文） 或 leaflets 放置在重點的位置
    
- 在人員產品示範 （英文） 或 booths
    
- 本機和遠端小組的線上事件
    
- 具有您贊助者主機鄉鎮館會議或傳送通知電子郵件
    
- 關於使用 Microsoft Search 共用自助支援文件
    
所有通訊應使用者都教育如何可以登入 Bing 來尋找工作結果
  
 **連絡夥伴小組**
  
組成 365 之一部分的 Microsoft，Microsoft Search 遵守 GDPR 和層 C 的指導方針。有些組織可能需要與您的隱私及安全性小組共用 Microsoft 搜尋的相關資訊。如需詳細資訊，請參閱[Microsoft 搜尋產品的安全性](security.md)。
  
請確定您的服務台知道相關的特性與功能的 Microsoft Search，以及使用者需求，才可登入，讓他們可以更有效率地支援使用者。如需詳細資訊，請參閱[使用 Microsoft 搜尋](use/about-microsoft-search.md)與[常見問題集](faqs.md)。
  
 **利用試驗使用者評估**
  
開始使用一小群使用者評估影響的 Microsoft Search 及收集早期的意見反應。這讓您有時間變更的內容、 更新通訊，並設定廣泛導入的期望。搜尋的進階使用者或早期採用者皆為這更好應徵者。
  
若要協助尋找試驗使用者：
  
- 合作夥伴與組織內業務發起人
    
- 識別一群早期採用者
    
- 識別執行大量的企業搜尋的群組
    
- 識別其他的地理位置，包括國際中的使用者
    
詢問您的試驗使用者來提供意見反應選項在 Microsoft 搜尋或使用透過簡單問卷的輸入。
  
若要要求調查問題： 
  
- 您是否了解如何登入和使用 Microsoft Search？
    
- 它的運作如預期般運作方式
    
- 您想要哪些功能？
    
- 已能夠找到工作結果吗？
    
- 工作結果很有幫助嗎？
    
- 關於 Microsoft 搜尋清楚且容易理解是通訊吗？
    
這是更好的時間開始測量效益的階段 2 中所識別的評量。
  
當您啟動的前導階段中時，請考慮使用更廣泛的組織共用 Microsoft Search 偷預覽。
  
## <a name="stage-4-roll-out"></a>階段 4： 首次

每一個組織有不同思考產品部署。以下是一些考量的事項。
  
 **廣泛部署**
  
當您準備好時，允許所有使用者的 Microsoft 搜尋貴組織存取或使用的分時期的方法來導入多緩慢。 
  
 **產生傳達方式**
  
建立樂趣關於 Microsoft 搜尋組織中：
  
- 編列試驗使用者來提供對等協助及專業知識。
    
- 遵循上通訊和您規劃在階段 3 中的事件。
    
 **進行中的通訊**
  
第一個幾週和月維護節奏持續的以利與 Microsoft Search 參與的通訊。這會通知使用者，而且會請新的使用者。每個通訊應說明 Microsoft Search 可以幫助他們的工作。
  
傳送功能及搜尋，以嘗試，請反白顯示的持續進行通訊，以及使用者成功案例。
  
 **收集意見反應及改善**
  
讓他們知道您注意聆聽它們並在其意見反應做且支援使用者。
  
若要查看如何執行產品以及內容使用前瞻儀表板。利用此資訊來更新書籤、 Q&As，以及其他通訊。這也可協助您：
  
1. 改善內容品質： 檢閱標題、 描述和 Url，最佳化關鍵字、 移除過時的資訊，並新增新的資訊
    
2. 判斷哪些功能最常 — 或常 — 使用
    
3. 進行調查、 焦點群組及意見反應工作階段來了解使用者想法的 Microsoft Search、 增加上手，並識別改進區域
    
教育及意見反應是反覆執行的程序。可能需要進行變更的內容及為您的使用者通訊採用 Microsoft 搜尋。
